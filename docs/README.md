# OctoAcme Project Management Documentation

Welcome! This repository collects the core project management process documentation for OctoAcme. The README summarizes our project management approach and provides links to all detailed guides below.

## 🚀 Where to Start (Onboarding)

| I am… | Start here |
|-------|-----------|
| **New to the team** | Read the [Project Management Overview](octoacme-project-management-overview.md) and [Roles and Personas](octoacme-roles-and-personas.md) |
| **Starting a new project** | Follow the [Project Initiation Guide](octoacme-project-initiation.md) and set up your [RACI Matrix](templates/raci-matrix-template.md) and [Decision Log](templates/decision-log-template.md) |
| **In planning** | Use the [Project Planning Guide](octoacme-project-planning.md) and agree on [Definition of Ready & Done](templates/definition-of-ready-and-done.md) |
| **In execution** | Reference [Execution & Tracking](octoacme-execution-and-tracking.md) and send [Weekly Status Updates](templates/weekly-status-update-template.md) |
| **Approaching a release** | Complete the [Release Readiness Checklist](templates/release-readiness-checklist.md) |
| **Closing a project** | Run a [Retrospective](octoacme-retrospective-and-continuous-improvement.md) and complete the [Project Closeout Checklist](templates/project-closeout-checklist.md) |

## Summary of OctoAcme Project Management Processes

- Customer-first focus, iterative delivery, and clear ownership
- Emphasis on data-informed decision making and psychological safety
- Defined roles (Project Manager, Product Manager, Developer, etc.)
- Lifecycle: Initiation → Planning → Execution → Release → Retrospective
- Key artifacts: Project One-pager/Charter, Backlog, Roadmap, Definition of Ready/Done, Risk Register, RAID Log, Decision Log, Retros, and Action Items
- Cadence: Regular standups, syncs, demos, and retrospectives
